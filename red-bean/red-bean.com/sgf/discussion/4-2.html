About the ST[] property: the draft states: Basically most programs show variations in two ways: as (A) markup on the board (if the variation contains a move) and/or as (B) a list (in a seperate window). (A) is done e.g. by xgoban (B) is done e.g. by MGT It continues with: The style number consists two options. 1) show variations of successor node (childs) (value: 0) show variations of current node (siblings) (value: 1) affects markup & list 2) do board markup (value: 0) no (auto-) board markup (value: 2) affects markup only. The final number is calculated by adding the values of each option.
