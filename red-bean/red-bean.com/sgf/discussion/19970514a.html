Date: Wed, 14 May 1997 19:19:42 +0200 (MET DST) From: Arno Hollosi Subject: SGF - discussion closed / specification ready Dear SGFers, I've uploaded the last version of the SGF FF[4] spec and plan to make this specification public in about 10 days. You've got time until then to read through the draft again and email me any errors I've made. Note: I don't want to discuss any topics we've solved in the past. If you've comments about the few changes then feel free to email them. Summary of changes: =================== - faulty game-info properties may remain where they are - converting linebreaks to LF no longer part of spec - new property PM: for printing - changed FG property: may contain diagram name and print flags - rewritten DT spec: partial dates and shortcuts for multiple dates - CA property spec extended: names should be the same as used in MIME messages. - updated FF4 example to latest spec - uploaded a MS-DOS executable of SGFC V0.6 (a new version which includes all changes of the SGF spec will be released when the spec is made public) - new index: all properties from FF[1] to FF[4] - some minor changes I can't remember now - suggested how to number nodes (start with zero) To be included: two more example files which illustrate the use of FG, MN and PM for creating printable diagrams. I've worked on SGF FF[4] (spec, example files, SGFC) for about a year now. Everything started when I thought it would be nice to have some arrows (e.g. for displaying influence) :-) It turned out to be more work than I expected. I want to thank all of you who have participated in the past and invested time helping to create this standard. I hope that coders of SGF applications make them a little bit more compatible than in the past - so that SGF becomes what it's meant to be: a standard for interchanging game records. I suggest we keep the email list some time for discussing implementation issues or other SGF related topics. /Arno
