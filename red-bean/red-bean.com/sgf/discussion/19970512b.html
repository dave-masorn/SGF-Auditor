Date: Mon, 12 May 1997 00:31:30 -0400 From: Dave Dyer Subject: Re: linebreaks... >> >> SGF is NOT intended to be human readable! >> No fiddling with text editors should be necessary - if you do use a text >> editor then use one able to understand single LF linebreaks. I disagree with this. One of the virtues of SGF is that it is both human and machine readable, so it is feasable to use text editors to fix flaws or add frills in ways that individual reader/writer programs might not accomodate. On the subject of line breaks, I believe the only acceptable statement is that readers should treat any of (cr, lf, crlf) as a single line break, and writers should use one style consistantly. It is unreasable to try to legislate that one partucular style be used when the three camps all use a different style, and none is likely to change. --- My home page: http://www.andromeda.com/people/ddyer/home.html
