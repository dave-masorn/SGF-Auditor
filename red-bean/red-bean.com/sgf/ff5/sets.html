Property sets

The basic idea is to group properties into sets. Properties of a set have a common feature, such as function or suitability for a certain task. This is a convinient help for programmers to decide which properties their program should support.

Properties may belong to more than one set. Core sets must not be game-specific.

How should sets be defined? By ...

  • function, e.g. board-markup, timing, game-info, moves, annotations, ...
  • task, e.g. "Needed-To-Replay-Games", "Needed-To-Print-Games", ...
  • type, e.g. core sets, optional sets, game-specific sets, private sets, ...
